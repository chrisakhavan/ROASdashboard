# Crypto Casino ROAS Dashboard

A beautiful, interactive dashboard for tracking Return on Ad Spend (ROAS) across multiple marketing channels.

## Files Included

- `index.html` - Main HTML file
- `app.js` - Dashboard application logic

## Deployment Instructions

### Option 1: GitHub Pages (Recommended)

1. **Delete your current repository** or create a new one
   - Go to https://github.com/chrisakhavan/ROASdashboard/settings
   - Scroll to bottom and click "Delete this repository"
   - OR create a new repo with a different name

2. **Create a fresh repository:**
   - Go to https://github.com/new
   - Name it: `roas-dashboard`
   - Make it **Public**
   - Do NOT add README, .gitignore, or license
   - Click "Create repository"

3. **Upload files:**
   - Click "uploading an existing file"
   - Drag and drop BOTH files: `index.html` and `app.js`
   - Click "Commit changes"

4. **Enable GitHub Pages:**
   - Go to Settings → Pages
   - Source: "Deploy from a branch"
   - Branch: "main" → "/ (root)"
   - Click "Save"

5. **Wait 2-3 minutes**, then visit:
   `https://chrisakhavan.github.io/roas-dashboard/`

### Option 2: Local Testing

1. Download both files to the same folder
2. Double-click `index.html` to open in your browser
3. Dashboard should load immediately

## Features

- ✅ 5 Marketing Channels: Paid Media, Gambling Streamers, Sports Cappers, Influencer Affiliates, Traffic Affiliates
- ✅ Time Period Views: Day 7, 30, 90, and 180 ROAS projections
- ✅ Filters: Country and Device Type (Web vs Mobile)
- ✅ Interactive Charts: Trend lines and bar charts
- ✅ Cohort Analysis: Week-by-week performance matrix
- ✅ Real-time Metrics: Trending indicators and comparisons

## Troubleshooting

If the page shows blank:
1. Make sure BOTH files are in the same directory
2. Check browser console (F12) for errors
3. Wait 3-5 minutes after pushing to GitHub for Pages to build
4. Clear browser cache and hard refresh (Ctrl+Shift+R)

## Support

If you have issues, check:
- Both files are uploaded to GitHub
- Files are in the root directory (not in a folder)
- GitHub Pages is enabled in Settings
- You're accessing the correct URL
